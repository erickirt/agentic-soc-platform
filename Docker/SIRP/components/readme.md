存储 SIRP 用到的自定义展示组件代码

Nocoly 低代码平台的 React 自定义组件开发

1. 官方核心架构规范

- 组件明确区分为 "存储类型" 和 "引用类型".
- 存储类型: 基于现有控件支持存储值, 接收 `value` (当前字段值) 和 `onChange` (调用以更新值) 属性.
- 引用类型: 仅用于呈现已有数据, 自身无法存储值. 接收 `formData` (格式为 { [controlId]: control }), `env` (包含引用字段的别名
  env.name, 以及 isMobile, isDisabled 属性), `currentControl` (当前字段). 数据获取方式必须类似于
  `formData[env.别名]?.value`.
- 仅支持原生单函数 (Function Component) 写法, 所有逻辑必须在这个单一函数内实现.
- 绝对不允许使用任何 import 语句. React 下的所有属性 (如 useState, useEffect) 已暴露到组件内, 可直接原生使用.

2. UI 与样式规范

- 仅支持使用 Tailwind CSS 进行样式排版.
- 内置图标必须使用 `<LucideIcon name="IconName" size="16" />` 组件, name 采用大驼峰写法.
- 最外层容器绝对不能包含任何边距 (p-*, m-*) 和边框 (border, shadow 等), 以确保无缝融入系统画布.


3. 代码洁癖与错误处理

- 编写的代码绝对不能添加任何调试信息 (如 console.log) 或单元测试代码.
- 生成的代码中不允许添加任何注释 (除非我明确要求).
- 遇到数据解析失败或格式异常等情况, 直接通过 `throw new Error("...")` 抛出异常, 不需要渲染异常状态 UI.

4. 沟通与文本规范

- 回答或编写代码之前, 必须确认是否已经获取到足够的信息 (如组件是存储还是引用类型, 数据结构, 交互逻辑). 如果信息不足,
  必须向我提问, 不要猜测或者直接回答.
